# エラードキュメント

本ドキュメントでは、OpenABEのエラーハンドリングメカニズム、一般的なエラー、およびそのトラブルシューティング方法について説明します。

## 1. エラーハンドリングメカニズム
OpenABEは、エラー報告にC++の例外ベースシステムを使用しています。すべての暗号化例外は `MessageException` から派生しています。

### 例外階層:
* `std::exception` $\rightarrow$ `MessageException`: `.what()` メソッドを提供するベースクラスです。
    * $\rightarrow$ `CryptoException`: 一般的な暗号操作が失敗したときにスローされます（例：無効なパラメータ、復号失敗）。
    * $\rightarrow$ `ZCryptoBoxException`: ハイレベルラッパーAPI内で発生したエラーに特化したものです。

## 2. 一般的なエラーケースと原因

| エラー / 例外メッセージ | 原因 | 対応策 |
| :--- | :--- | :--- |
| **復号失敗 (Decryption Failure)** | `keyID` に関連付けられた秘密鍵が暗号文のアクセスポリシーを満たしていません。 | 秘密鍵に関連付けられた属性が暗号化ポリシーの要件を満たしているか確認してください。 |
| **無効なポリシー構文 (Invalid Policy Syntax)** | `encrypt()` または `keygen()` に渡されたポリシー文字列に構文エラーがあります（例：括弧の不足、無効な演算子）。 | ポリシー文字列のブール論理を確認してください（`AND`、`OR` 演算子を正しく使用しているか）。 |
| **鍵が見つからない (Key Not Found)** | 指定された `keyID` が内部キーストアに見つかりませんでした。 | 復号を試みる前に、正しいIDで `keygen()` が呼び出されていたか確認してください。 |
| **初期化エラー (Initialization Error)** | `InitializeOpenABE()` が失敗したか、APIを使用する前に呼び出されませんでした。 | アプリケーション起動時の最優先事項として `InitializeOpenABE()` を呼び出してください。 |
| **パラメータ不一致 (Parameter Mismatch)** | 暗号文が、現在復号に使用しているものとは異なるセットアップ/システムパラメータで暗号化されています。 | 暗号化者と復号者が同じシステムパラメータ（公開パラメータ）を使用していることを確認してください。 |
| **内部数学エラー (Internal Math Error)** | ペアリングや群演算中に基底の数学ライブラリ (ZML/RELIC) で失敗が発生しました。 | これは通常、暗号文が破損しているか、サポートされていないパラメータセットであることを示しています。 |

## 3. トラブルシューティングガイド

### シナリオ：「復号に失敗した」が属性は正しいはずである
* **形式の確認**: 鍵生成時に、属性リストがパイプ区切りの文字列（例：`"|attr1|attr2|"`）として渡されているか確認してください。
* **大文字小文字の区別**: OpenABEの属性は大文字小文字を区別します。`Attr1` と `attr1` は異なる属性として扱われます。
* **ポリシー論理の検証**: ポリシーが意図した論理に正しくマッピングされたブール式で記述されているか再確認してください（例：`(A AND B) OR C`）。

### シナリオ：アプリケーションが起動時にクラッシュする
* **ライブラリの確認**: すべての依存ライブラリ (RELIC, OpenSSL) が正しくインストールされ、ライブラリパス (`LD_LIBRARY_PATH`) に含まれているか確認してください。
* **初期化の検証**: 他のAPIコールよりも前に `InitializeOpenABE()` が呼び出されていることを確認してください。

## 4. エラーハンドリングのベストプラクティス
アプリケーションのクラッシュを防ぐ的に、すべてのハイレベルAPIコールを `try-catch` ブロックで囲んでください：
```cpp
try {
    ctx.decrypt(keyID, ciphertext, plaintext);
} catch (const CryptoException& e) {
    std::cerr << "暗号学的エラーが発生しました: " << e.what() << std::endl;
    // 回復処理またはユーザーへの通知を行う
}
```
